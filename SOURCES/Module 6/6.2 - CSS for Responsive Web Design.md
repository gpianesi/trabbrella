
**CSS** (*Cascading Style Sheets*) is a stylesheet language that allows developers to control the **visual presentation** of web documents.  
Its primary goal is to **separate content (HTML)** from **design and layout**, improving maintainability, flexibility, and reuse.

---

# CSS Syntax

## Core Concepts

- **Style sheet** – a collection of rules that define how selected elements should appear.
- **Selector** – targets HTML elements by **type**, **id**, **class**, or other criteria.
- **Declaration block** – contains one or more **property–value** pairs.
- **Property** – the aspect of the element to change (e.g., `color`, `font-size`).
- **Value** – the setting applied to the property.

## Example

```html
<style>
  h1 {
    color: black;
  }

  #content {
    border: 1px solid red;
  }

  div.normal {
    font-size: 16pt;
    color: blue;
  }
</style>
```

---

# Integrating CSS with HTML

CSS can be applied in three primary ways.

## Inline CSS

Styles are written directly inside an HTML element using the `style` attribute.

```html
<p style="font-size: 12px; color: #CCC;">Some text</p>
```

**Use case:** quick testing or small, one-off changes.  
**Limitation:** difficult to maintain for large projects.

---

## Internal CSS

Rules are placed inside a `<style>` element within the document’s `<head>`.

```html
<head>
  <style>
    p.normal {
      font-size: 12px;
      color: #CCC;
    }
  </style>
</head>
```

**Use case:** single-page websites.  
**Limitation:** styles are not shared across multiple pages.

---

## External CSS

Styles are stored in a separate `.css` file and linked from the HTML document.

```html
<!-- HTML document -->
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<<<

<<<css
/* styles.css */
p.normal {
  font-size: 12px;
  color: #CCC;
}
```

**Use case:** recommended for most projects.  
**Advantage:** reusable, scalable, easier to maintain.

# CSS and Responsive Design

**Responsive design** ensures that web content displays correctly on devices with different screen sizes and resolutions.

## Key Techniques

- **Media queries** – apply different style rules at defined **breakpoints** (screen widths).
- Modern layout systems such as **Flexbox** and **Grid** help create adaptable structures.

## Example

```css
/* Base styles for mobile devices */
.container {
  padding: 20px;
  font-size: 16px;
}

/* Media query for tablets */
@media (min-width: 768px) {
  .container {
    padding: 40px;
    font-size: 18px;
  }
}

/* Media query for desktops */
@media (min-width: 1024px) {
  .container {
    padding: 50px;
    font-size: 20px;
    max-width: 960px;
    margin: auto;
  }
}
```
